1) DNS (Domain Name System)

When a user enters https://ngocbnguyen.github.io in a browser, the browser must first determine the server’s IP address because computers communicate using IP addresses rather than domain names. The browser sends a DNS query to a DNS resolver, which searches the Domain Name System for the address associated with the domain. The resolver contacts DNS servers until it reaches GitHub’s authoritative DNS servers, which return the IP addresses used by GitHub Pages (such as 185.199.108.153–185.199.111.153).

2) IPv4 Addressing

The DNS result shows the IPv4 address for my domain. That IP address is the network destination the browser connects to. Once the IP address is returned, the browser establishes a connection to that server using HTTPS. The GitHub Pages server then responds with the website files, which the browser renders as the webpage. In my deployment, the resolved address is:

3) Protocols HTTPS

My website ngocbnguyen.github.io uses HTTPS (Hypertext Transfer Protocol Secure), which encrypts communication between the browser and the server using TLS. HTTPS ensures that the data exchanged between users and the website cannot be easily intercepted or modified.
When the browser requests the webpage, the server responds with the status code 200 OK, indicating that the request was successful and the page was delivered correctly. The browser sends request headers that include information such as the host, user agent, and accepted content types. The server responds with response headers that describe the returned content, such as the content type and caching policies. These headers and status codes can be viewed using the browser’s Developer Tools Network tab.
