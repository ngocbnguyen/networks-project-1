1) DNS (Domain Name System)

When a user visits my site using the domain name, the browser must first find the IP address. DNS does this by resolving the domain (example: username.github.io) to the server IP address that hosts the website.

Command I ran (evidence)

nslookup yourdomain.com

Replace yourdomain.com with your real domain.

3) Protocols (HTTP vs HTTPS) + Status Codes + Headers

After DNS resolution, the browser connects to the server using TCP/IP and requests the page using HTTP. For a public deployment, my site uses HTTPS, which is HTTP over TLS encryption.

How I collected evidence

  1. Open the deployed website
  2. Right click, choose Inspect
  3. Go to the Network tab
  4. Click the main document request (home page)

4) Security feature implemented

My site is served over HTTPS. This protects traffic in transit using encryption, so users are not sending page requests in plain text.
